Constructor

new Chart(elementId, config)

Конструктор инициализирует объект графика

Parameters:
Name Type Description
elementId object

id контейнера (DOM - елемент div) в котором будет отображаться график
config object

объект с настройками (конфигурацией) графика

Properties
Name Type Description
dataURL string

корневая папка с файлами таблиц архивных данных, этот URL должен быть в доступном для скачивания месте,как правило это каталог "/public" Например:"/logs" Внимание!!! Слеш в конце не ставить он добавляется автоматически !!!
startDate: string

имя файла таблицы данных, расширение ".log" добавляется автоматически Например: startDate="2021-01-06", таким образом URL для запроса таблицы будет: /logs/2021-01-06.log
y object

диапазон отображаемых значений по оси y (). Например: y:{min:0,max:1000}

Properties
Name Type Description
min Number

минимальное значение по оси y ; например 0
max Number

максимальное значение по оси y; например: максимально возможная температура в печи(Тномин+50С)
registers object

описание регистров

Properties
Name Type Description
id string

id регистра, точно такое же название какое используется в таблице архива .log Например: "1-T"
title string

имя регистра, понятное для человека, используется как title Например: "T1"
type string

тип значения.Например: "integer" (пока не используется)
units string

единицы измерения Например: "*С"
description string

подробное описание регистра Например: "Температура в верхней зоне"
legend string

используется для подписывания легенды в графике. Среднее по длине описание. Например: "Зона№1.Верх.Задание"

getData() → {null}

делает загрузку таблицы данных по URL: ${config.dataURL}+"/"+${config.startData}+".log" Парсит и сохраняет в свойстве data Файл с данными должен быть в виде текстового файла с разделителями TAB="\t" (tsv-формат) первая строка описывает имена регистров, последующие - данные, например: ''' time 1-T 2-T 3-T \n 2019-10-14T22:12:21 25 30 37 \n 2019-10-14T22:17:11 32 38 41 '''
